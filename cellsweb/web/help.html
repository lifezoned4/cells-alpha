Overview

There are two kinds of States: Energy (rectangular) and Cell (round).

Fields which are gray are empty fields

A cell can move from one field only into a empty field

A Registermaschine is used to dertmin which direction if any will be choosen

Your task will be to move the cell in some directions and after it has collected Energy fields of there compliment color it should start to reproduce

To do so you have to understand how the registermaschine works

GreenCode Registermachine

A Reigster maschine is a set of Operations combinded with a set of Registers. A Operation has the structure:

  • |OperandName| |OperandFlag||Operand:Number|;

    • A Example Operation would be LOAD #1; It loads the Constant number 1 into the ALU Register. ALU is the Register 0.

    STORE #8; stores the number which is in Register 0 into the Register at the number 8. This is a special Register. It determins in which direction the cell will be moving in the next model step.


    Exercise 1

    Try to start a cell (by selecting energy, putting code into the GreenCode TextBox and press start. Selecting can be done by clicking a rectangle field which is not gray). After started a cell you do it again but this time type something into the greencode field which will make the cell move.

    Soloution 1

    LOAD #3; STORE #8;

    The Content of Register 8 will be interpreted as so:

  • 0: dont move
  • 1: move north
  • 2: move east
  • 3: move south
  • 4: move west
  • 5: dont move
  • 6: move north

    As you can notice the field is moduleted by 5 and the a direction is choosen.

    After we learned how to move, we know shall undestand how eating works.

    When a green cells is near a blue cell or Energyfield it eats the other field. Same for blue near red. And red near green.

    Exercise 2

    Try to move a cell in a color to a color it can eat.

    To master Exercie 2 we learn know some operations to do more logic

    ADD #5; adds the number 5 to the ALU

    Exercise 3

    Move your cell in circles

    Solution 3

    ADD #1; STORE #8; ADD #1; STORE #8; ADD #1; STORE #8; ADD #1; STORE #8;

    Solution 3 alternativ

    ADD #1; STORE #8;

    The alternativ solution uses the registermachine behavior of roundrobbing the operations. When the end of the list is reached its starts at the top again
